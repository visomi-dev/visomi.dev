# Projects Section Redesign Prompt

## Full Prompt

```text
Redesign the projects section into a more editorial case-study style exploration.

Keep the existing page context, existing brand tone, and existing project content. Do not redesign the global shell, navigation, footer, or overall design system. Focus only on the projects section.

Direction:
Transform the current card grid into a sequence of rich project story blocks. Each project should feel like a product case study preview instead of a portfolio card.

Layout:
For each project, create a two-column composition on desktop:
- Left side: project narrative and decision-making content
- Right side: large visual previsualization with supporting image navigation or image stack

On mobile:
- Stack vertically
- Text first, image second
- Keep strong spacing and readability

For each project block include:
- Project title
- Role, year, and category
- Short summary
- A small "decisions made" area explaining key product or technical choices
- A compact list of technologies
- A set of image-related descriptions:
  each screenshot should have a short caption or explanation of what that screen shows, why it matters, and what part of the flow it represents

Content style:
The left column should read like:
- what the product is
- what problem it solves
- the important parts of the flow
- the design or engineering decisions made for that project

The right column should feel like:
- a polished project preview
- one main image visible at a time
- optional thumbnail selector, stepped gallery, or layered preview cards
- every image should feel tied to the explanation on the left

Visual direction:
- More premium and narrative than dashboard-card based
- Clean, modern, confident, product-designer + engineering portfolio style
- Strong hierarchy, elegant spacing, calm typography, subtle accents
- Avoid generic startup landing page patterns
- Avoid making it look like a marketplace or ecommerce page

Interaction ideas to explore:
- Hovering or selecting a screenshot updates the explanation text
- A vertical timeline or labeled steps for each product flow
- Sticky text column while the image gallery changes
- Smooth transitions between screenshots
- Clear "why this screen matters" microcopy

Important constraints:
- Do not change the rest of the page layout outside the projects section
- Preserve the portfolio nature of the page
- Keep the structure suitable for multiple projects, not just one
- Make it realistic for a developer portfolio with fintech, platform, and internal tools
- Respect that some projects may have private previews or fewer screenshots

Design goal:
Make the projects section feel like a thoughtful product walkthrough where each project shows both the interface and the reasoning behind the implementation decisions.
```

## Short Prompt

```text
Explore a redesign of the projects section only. Keep the existing page context and visual language, but transform the current project cards into editorial case-study blocks.

For each project:
- left column: title, role, year, category, short summary, key decisions made, technologies, and short explanations of each screen/part of the flow
- right column: large image previsualization with supporting thumbnails or a layered gallery

The section should feel premium, narrative, and product-focused, like a portfolio case study preview. Avoid redesigning the shell, navbar, footer, or design system. Focus only on the projects section. Keep it responsive, with text first and image second on mobile.
```
